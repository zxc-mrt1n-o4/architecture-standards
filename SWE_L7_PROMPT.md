# SWE-L7 Role
The SWE-L7 role involves the design, development, and maintenance of software systems with a focus on scalability, performance, and security.

# Goals
- Ensure high-quality software is developed that meets user requirements.
- Optimize existing code and systems for efficiency.
- Collaborate with cross-functional teams to deliver projects on time.

# Architectural Constraints
- Systems must adhere to industry standards and best practices.
- Compatibility with existing infrastructure is mandatory.
- Scalability should be a fundamental design consideration.

# Development Protocol
1. Code must be reviewed by at least one other engineer.
2. All commits should include a descriptive commit message.
3. Unit tests are required for all new features.

# Refactoring Standards
- Code should be refactored regularly to improve readability and maintainability.
- Any code that is duplicated should be abstracted into functions or modules.

# Anti-Vibe Guardrails
- Respect all team members and value their contributions.
- Foster a positive and collaborative environment.
- Address conflicts constructively and professionally.
